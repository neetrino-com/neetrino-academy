# Настройка Cloudinary для загрузки файлов

## Проблема
По умолчанию система загрузки файлов настроена на использование Cloudinary, но переменные окружения не настроены, поэтому файлы сохраняются локально.

## Решение

### Вариант 1: Настройка Cloudinary (рекомендуется)

1. Зарегистрируйтесь на [Cloudinary.com](https://cloudinary.com)
2. Получите данные из Dashboard:
   - Cloud Name
   - API Key
   - API Secret

3. Обновите файл `.env`:
```env
# File Upload (Cloudinary)
CLOUDINARY_CLOUD_NAME="ваш-cloud-name"
CLOUDINARY_API_KEY="ваш-api-key"
CLOUDINARY_API_SECRET="ваш-api-secret"
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME="ваш-cloud-name"
```

4. Создайте Upload Preset в Cloudinary:
   - Перейдите в Settings > Upload
   - Создайте новый preset с именем `academy_uploads`
   - Установите Signing Mode: "Unsigned" (для простоты)

### Вариант 2: Использование локального хранения (текущее состояние)

Система автоматически использует локальное хранение файлов в папке `public/uploads/temp/` если Cloudinary не настроен.

**Ограничения локального хранения:**
- Файлы доступны только на том же сервере
- Нет автоматического сжатия изображений
- Нет CDN для быстрой загрузки
- Файлы могут быть потеряны при перезапуске сервера

## Проверка работы

После настройки Cloudinary:
1. Перезапустите сервер разработки
2. Попробуйте загрузить файл в builder'е курсов
3. Проверьте, что файлы загружаются в Cloudinary Dashboard

## Логи

Если возникают ошибки, проверьте:
1. Консоль браузера для ошибок клиента
2. Логи сервера для ошибок API
3. Cloudinary Dashboard для статуса загрузок
